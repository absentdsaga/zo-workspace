## Architecture

**Single source of truth:** `Documents/VURT-master.md`
All VURT institutional knowledge lives in one place. Deep-dive annexes live in `references/` for specialized topics.

## Usage

1. **Always start by reading `Documents/VURT-master.md`** — this is the master doc with everything: company, team, brand, content, markets, audience, technical platform, social state, competitive landscape, production pipeline, workstreams, and decision log.
2. **Go deeper with reference annexes** when a task needs specialized detail:

### Reference Annexes
- `references/tracking-analytics.md` — Pixel tracking plan (15 pixels, 3 phases), GA4 event mapping, UTM structure, user journey funnel, paid ads team technical requirements
- `references/competitive-intel.md` — Detailed competitor profiles: ReelShort, DramaBox, Watch Club, PineDrama, major studio experiments, VURT positioning
- `references/digital-presence-audit.md` — Full March 2026 audit: every website page, Instagram, TikTok, YouTube, Facebook, Snapchat, app stores
- `references/key-links.md` — All platform URLs, social handles, press links, internal docs, team contacts

### When to activate
- Any VURT copy, strategy, pitch, or marketing task
- Competitive analysis or market research
- Brand voice work (must follow brand bible guidelines in master doc)
- Content pipeline or social media production work
- Investor/press materials
- Pixel tracking, analytics, GA4, or paid ads attribution work
- Technical platform questions (URL structure, Enveu constraints, share links)
- SEO implementation or technical audit discussions

### Keeping it current
- When new decisions, data, or status changes come up in any VURT conversation, **update `Documents/VURT-master.md` directly** — not a separate file.
- Update reference annexes only when their specialized topic area changes.
- Never duplicate master doc content into reference files.
