## Changes for v2.2

### 1. Raise Minimum Confidence: 45 → 70
```typescript
// Current
private readonly MIN_SMART_MONEY_CONFIDENCE = 45;

// New
private readonly MIN_SMART_MONEY_CONFIDENCE = 70;
```

**Rationale:**
- Current v2.1: Conf 50-60 has 0% WR (4 losses, 0 wins)
- Archive Master: Conf 60 was good (61.9% WR), but current market is different
- Archive Master: Conf 45-55 averaged only 36-38% WR
- **Safe bet:** Start at 70, can lower to 60 later if needed

**Expected impact:**
- Fewer qualifying opportunities (maybe 5-8 instead of 13)
- Higher quality trades
- Better avg P&L per trade

### 3. Reduce Max Concurrent Positions: 7 → 5
```typescript
// Current
private readonly MAX_CONCURRENT_POSITIONS = 7;

// New
private readonly MAX_CONCURRENT_POSITIONS = 5;
```

**Rationale:**
- 13 opportunities per scan, only need 7 → poor selectivity
- Lower positions = more selective (only best opportunities)
- Better risk management (less capital at risk)
- Archive Master may have used fewer positions (unknown)

**Expected impact:**
- Not always maxed out (more breathing room)
- More selective entries (top 5 instead of top 7)
- Lower drawdown risk

### 4. Add "Skip Previous Losers" Filter
```typescript
// In paper-trade-bot.ts, line ~304 (in availableOpps filter)
const availableOpps = qualified
  .filter(opp => {
    // Existing: Skip if currently holding
    const isOpen = this.trades.some(t => t.status === 'open' && t.tokenAddress === opp.address);

    // Existing: Skip if blacklisted
    const isBlacklisted = this.ruggedTokens.has(opp.address);

    // NEW: Skip if previously lost
    const previouslyLost = this.trades.some(t =>
      t.tokenAddress === opp.address &&
      t.status === 'closed_loss'
    );

    if (previouslyLost) {
      console.log(`   ⏭️  SKIPPED: ${opp.symbol} (Previously lost on this token)`);
    }

    return !isOpen && !isBlacklisted && !previouslyLost;
  })
  .sort((a, b) => b.score - a.score);
```

**Rationale:**
- Current v2.1: Repeating losers cost -0.06 SOL (CvVncV trades 2-5, 3vgJGbBD trades 2-3)
- Would have saved ~0.09 SOL on current run alone
- 3-loss blacklist prevents trades 4-6, but we should prevent trade 2!
- Archive Master's LEO exception (1W/2L but profitable) was 1 outlier in 288 trades

**Expected impact:**
- Prevent repeat losses after first loss
- Save ~0.04-0.09 SOL per repeated loser avoided
- Forces bot to find new opportunities instead of re-trading losers

### 5. Add Nameless Token Filter
```typescript
// In paper-trade-bot.ts, after getting opportunity
if (!opp.symbol || opp.symbol.trim() === '' || opp.symbol === '‎') {
  console.log(`   ⏭️  SKIPPED: ${opp.address.slice(0, 8)}... (No symbol/name)`);
```

**Rationale:**
- Current v2.1: Both blacklisted tokens were nameless ("‎")
- Nameless token 1: -0.0722 SOL (3 losses)
- Nameless token 2: -0.0078 SOL (1W/4L)
- Hard to track, usually sketchy projects

**Expected impact:**
- Avoid low-quality tokens
- Better visual tracking in logs
- Estimated savings: 0.05-0.10 SOL per run

1. Reset to fresh state (0.5 SOL)
2. Let run for 50+ trades or 24 hours
3. Compare metrics vs v2.1:
   - Win rate (target: >35%)
   - Avg P&L per trade (target: >+0.01 SOL)

   - Positions used (expect 3-5/5 instead of 7/7)

- Win rate >35% (Archive Master: 38.9%)
- Total P&L positive after 50 trades
- No nameless tokens traded
- No conf 80 tokens traded

## Rollback Plan

If v2.2 performs worse than v2.1:

1. Keep min confidence at 70 (this is solid)
2. Re-enable conf 80 (maybe it was market conditions)
3. Raise max positions back to 7 (more opportunities)
4. Remove "skip previous losers" filter (allow comebacks like LEO)
5. Remove nameless filter (if causing too many skips)

**Incremental tuning:**
- If too few trades: Lower min conf 70 → 65
- If still maxed out: Lower max positions 5 → 3
- If missing good repeats: Only skip tokens with 2+ losses (not 1)
