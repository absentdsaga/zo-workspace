Methodology

- Data source: Google Analytics 4 (Property ID 518738893, myvurt.com)

- Video data: Mux Data API (views, rebuffer %, playback failures, completion, startup time). Migrated from NPAW on 2026-04-22 — Mux is the new source of truth. videoTitle/videoSeries metadata is not yet populated in the player, so top-content tables show video IDs; once devs wire those fields, titles will resolve to show names.

- Landing page queries use landingPage (path only, no query strings) to prevent fbclid/utm parameter fragmentation from causing GA4 to drop rows. Cross-dimension totals are verified against single-dimension totals.

- Engagement Rate: % of sessions lasting >10s, with 2+ page views, or a key event (GA4 standard definition)

- Bounce Rate: % of sessions that were NOT engaged (inverse of engagement rate)

- Geo Efficiency Score: engagement rate x sessions. Surfaces high-volume/low-engagement geos (ad waste) vs high-quality traffic.

- Daily Snapshot freshness: Automatically uses yesterday's data when GA4 processing is complete (engaged sessions >= 10% of 7-day avg). Falls back to 2-days-ago when data is still processing.

- Social media data: Instagram via Meta Graph API (IG Business Account 17841479978232203), YouTube/TikTok/X via public profile scraping

- All numbers are pulled directly from APIs — no manual adjustments or estimates except where explicitly labeled

- Health Score is a weighted composite: Growth (25), Engagement (25), Retention (25), Traffic Quality (25)

- '(not set)' entries indicate GA4 tracking gaps where screen names aren't being passed in analytics events
